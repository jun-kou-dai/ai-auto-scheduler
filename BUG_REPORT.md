# バグ相談: 提案画面でタスクが「未割当」になり配置されない

## 症状

ユーザーが「今日の9時からトレーニングをします」と入力すると、提案画面で以下が表示される:

- 「未割当タスク（N件）」
- 「締切り（2026/2/14）まで十分な連続空き時間はありません。」

**仕様ではダブルブッキングしてでも必ず全タスクを配置する**はずだが、配置されていない。

## 謎

「締切り（2026/2/14）まで十分な連続空き時間はありません。」というメッセージが**ソースコード内に存在しない**。

```bash
grep -r "十分な" src/         → ヒットなし
grep -r "連続空き" src/       → ヒットなし
grep -r "未割当タスク" src/   → ヒットなし
```

ビルド済みバンドル (`dist/`) にも存在しない。

## 配信構成

```
npx expo export --platform web → dist/ にバンドル出力
node serve.js                  → dist/ を localhost:8081 で配信
```

- `dist/` のバンドルは **2026-02-13 22:59 UTC** に生成された古いもの
- ソースコード (`src/`) はその後修正済みだが、**リビルド `npx expo export` していない**
- つまりユーザーが見ているのは古いバンドルの可能性が高い

## しかし矛盾がある

- タイムゾーンバグ（「今日」が前日になる問題）を `src/` で修正した後、ユーザーの画面で締切日が 2/13 → **2/14 に変わった**
- 古い `dist/` バンドルが配信されているなら、タイムゾーン修正は反映されないはず
- **なのに日付は修正が反映されている** → Expo dev server も並行して動いている？ or バンドルが部分的に更新された？

## スケジューラのコード（scheduler.ts）

全タスクを必ず配置する設計:

```
Pass 1: 締切内の空きスロットに配置
Pass 2: 締切を無視して空きスロットに配置 + 警告
Pass 3: 空きがなくてもダブルブッキングで強制配置 + 警告
```

`generateProposal()` は常に `{ events: [...全タスク...], unassigned: [] }` を返す。

## ProposalScreen のコード

現在のソースコードでは `proposal.unassigned` を表示するUIが**存在しない**。`proposal.events` のみ表示している。

## 型定義

```typescript
interface Proposal {
  events: ProposalEvent[];
  unassigned: UnassignedTask[];  // { taskId: string; reason: string }
}
```

## Git履歴の手がかり

以前のコミットに以下がある:
```
3643ab5 fix: 締切超過時のフォールバック配置 - 未割当ではなく最短空き枠に配置+警告表示
0439c9a fix: スケジューラー根本改修 - 常に全タスク配置、手動時刻変更対応
```

→ **以前は「未割当」にする古いスケジューラが存在した**。これが `dist/` に残っている可能性。

## 考えられる原因の仮説

1. **dist/ のリビルド忘れ**: ソース修正が dist/ に反映されていない → `npx expo export --platform web` が必要
2. **Expoキャッシュ**: `.expo/` や Metro バンドラーのキャッシュに古いモジュールが残っている
3. **Expo dev server とserve.js の混在**: 両方が動いていて、AIサービス（ai.ts）はdev server経由で最新コードを使い、UI（ProposalScreen）は古いdist/から配信されている

## 環境

- Expo SDK 54, React Native Web
- サーバーのタイムゾーン: UTC（日本時間 -9時間）
- ユーザーのタイムゾーン: JST (Asia/Tokyo)
